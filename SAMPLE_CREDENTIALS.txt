================================================================================
                            IMPORTANT NOTES
================================================================================

DATA STORAGE:
  - All user data is stored in browser localStorage
  - Data persists across page refreshes
  - Clearing browser cache will reset all data
  - This is a demo environment - data is not permanent

PASSWORD SECURITY:
  - Passwords are hashed using Base64 (demo only)
  - In production, use bcrypt or similar strong hashing
  - Never share credentials in production environment

TESTING TIPS:
  - Use different accounts to test different user roles
  - Try the back button in admin dashboard
  - Test responsive design on mobile browsers
  - Switch between light and dark themes (top right)

RESETTING DATA:
  - To reset all data, clear browser localStorage
  - Or manually delete 'pulserelief_users' and 'pulserelief_staff' keys
  - Sample accounts will be recreated on next page load

================================================================================
                         FOR DEVELOPERS
================================================================================

To add more sample data:
  - Edit auth-system.js
  - Modify initializeSampleUsers() or initializeSampleStaff()
  - Add new user/staff objects to the arrays
  - Save and reload the page

Authentication System:
  - Uses localStorage for demo
  - File: auth-system.js
  - Class: AuthSystem
  - Methods: createUser(), loginUser(), loginStaff(), loginAdmin()

To view stored data in console:
  JSON.parse(localStorage.getItem('pulserelief_users'))
  JSON.parse(localStorage.getItem('pulserelief_staff'))
